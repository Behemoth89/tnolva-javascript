## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, field-to-label gap |
| md | 16px (`gap-4`, `p-4`) | Default element spacing, form field gaps |
| lg | 24px (`gap-6`, `p-6`) | Section padding, card internal padding |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps, card-to-edge spacing |
| 2xl | 48px (`gap-12`) | Major section breaks (form title to fields) |
| 3xl | 64px (`gap-16`) | Page-level spacing |

Exceptions:
- Touch targets minimum 44px height for form inputs and buttons (accessibility requirement)
- Submit button vertical padding: 12px (`py-3`) for comfortable click area

**Source:** Standard 8-point scale. Existing placeholder pages use `mt-2` (8px) for subtitle spacing.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-zinc-950` (#09090B) | Full-page background for auth pages |
| Secondary (30%) | `bg-zinc-900` (#18181B) | Card surface, navbar background |
| Accent (10%) | `text-amber-500` / `bg-amber-500` (#F59E0B) | Primary CTA buttons, active links, focus rings |
| Destructive | `text-red-500` / `bg-red-500` (#EF4444) | Error text, destructive actions only |

Accent reserved for:
- Primary submit buttons (Login, Register)
- Link text ("Don't have an account? Register")
- Focus ring outlines
- Active/selected states

**Semantic color usage:**
- Success states: `text-green-500` (future use, not in this phase)
- Warning states: `text-yellow-500` (future use, not in this phase)
- Disabled: `bg-zinc-800 text-zinc-600 cursor-not-allowed`
- Borders: `border-zinc-800` for subtle card/input borders
- Text primary: `text-zinc-100` for headings and body text
- Text muted: `text-zinc-400` for subtitles, placeholders, secondary text

**Source:** Dark theme with gold accent. Zinc palette provides warm, rich dark tones (not pure black). Amber-500 gold accent pairs naturally with zinc's warm undertones.

## Interaction Contracts

### Form Validation Flow
1. User focuses field → no validation
2. User blurs field → validate on blur (D-04)
3. Validation fails → show red error text below field
4. User types in field → clear error for that field (D-12)
5. Password confirmation validates against password value on blur (D-05)

### Submit Flow
1. User clicks submit → button shows spinner + disabled state (D-13)
2. API call in progress → button remains disabled
3. Success → redirect to `/dashboard`
4. API error → show inline field-level error (D-10) or generic network error (D-11)
5. Error clears when user types in affected field (D-12)

### Protected Route Flow
1. User visits `/dashboard` without auth → redirect to `/login` (D-06, D-07)
2. Return path saved → redirect back after login (D-07)
3. Authenticated user visits `/login` or `/register` → redirect to `/dashboard` (D-08)
4. Page reload → loading spinner shown during rehydration (D-09)

### Logout Flow
1. User clicks "Logout" in navbar → `clearAuth()` called (D-16)
2. Immediate redirect to `/login` (D-16)
3. Zustand persist clears from localStorage
