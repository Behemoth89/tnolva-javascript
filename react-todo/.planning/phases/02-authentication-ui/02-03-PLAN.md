Add `isLoading` state to the Zustand auth store to track persist rehydration.

**Changes to src/stores/useAuthStore.ts:**
1. Add `isLoading: true` to the initial state (starts true, persist sets it to false after rehydration)
2. Configure persist middleware with `onRehydrateStorage` callback:
   ```typescript
   persist(
     (set) => ({
       isLoading: true,
       // ... existing state
       setAuth: (payload) => set({ ...payload, isAuthenticated: true, isLoading: false }),
       clearAuth: () => set({ token: null, refreshToken: null, firstName: null, lastName: null, isAuthenticated: false, isLoading: false }),
     }),
     {
       name: 'auth-storage',
       onRehydrateStorage: () => (state) => {
         if (state) {
           state.isLoading = false;
         }
       },
     }
   )
   ```
3. Also update AuthState interface in src/types/auth.ts to include `isLoading: boolean`

**Why this matters (per D-09):** Zustand persist rehydrates asynchronously from localStorage. Without `isLoading`, there's a flash where `isAuthenticated` is false even though the user has a saved session. The ProtectedRoute component will check `isLoading` and show a spinner instead of redirecting to login during rehydration.

**What to avoid:**
- Do NOT change the persist `name` — it's 'auth-storage' and existing sessions depend on it
- Do NOT remove any existing fields or methods
