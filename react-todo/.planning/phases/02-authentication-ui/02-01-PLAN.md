Replace the placeholder LoginPage with a full login form component.

**Dark theme per UI-SPEC (overrides D-01 gray palette):**
- Outer: `min-h-screen flex items-center justify-center bg-zinc-950`
- Card: `bg-zinc-900 border border-zinc-800 rounded-lg p-8 w-full max-w-md`
- Title: "Sign In" centered, `text-2xl font-bold text-zinc-100 mb-6`
- Text: `text-zinc-100` for labels and body, `text-zinc-400` for placeholders/muted text
- Input borders: `border-zinc-800 bg-zinc-900 text-zinc-100 placeholder-zinc-400`
- Submit button: `bg-amber-500 hover:bg-amber-400 text-zinc-950 font-medium` (accent color per UI-SPEC)
- Link text: `text-amber-500 hover:text-amber-400`
- Error text: `text-sm text-red-500`
- Disabled button: `bg-zinc-800 text-zinc-600 cursor-not-allowed`

**Fields (per D-04 — inline validation on blur):**
1. Email field: `type="email"`, label "Email", placeholder "your@email.com", `min-h-[44px]` (accessibility touch target)
2. Password field: `type="password"`, label "Password", placeholder "••••••••", `min-h-[44px]`
3. Submit button: "Sign In", full-width, `py-3` (per UI-SPEC spacing)
4. General error displayed at top of form (below title, above fields) when `errors.general` is set

**State management:**
- `email`, `password` — controlled inputs via useState
- `errors` — object `{ email?: string, password?: string, general?: string }`
- `isSubmitting` — boolean
- `useNavigate` from react-router for redirect

**Validation on blur (per D-04):**
- Email: required, must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` regex
- Password: required, minimum 6 characters
- On blur: if field has been touched and value is invalid, set error message
- On change: clear the specific field's error (per D-12)

**Submit handler:**
1. Prevent default, validate all fields, if invalid return early
2. Set `isSubmitting = true`, clear all errors
3. Call `apiClient.post<LoginResponse>('/auth/login', { email, password })`
4. On success: call `useAuthStore().setAuth({ token, refreshToken, firstName, lastName })` from response, then `navigate('/dashboard')`
5. On 4xx error: set `errors.password` to "Invalid email or password. Please try again." (per UI-SPEC copywriting)
6. On network error: set `errors.general` to "Unable to connect. Please check your connection and try again." (per D-11, UI-SPEC)
7. On any error: set `isSubmitting = false`

**Submit button state (per D-13):**
- Disabled when `isSubmitting` is true
- Shows spinner when submitting: inline SVG with `animate-spin h-5 w-5 mr-2`
- Button text: "Signing in..." when submitting, "Sign In" otherwise

**Accessibility (per UI-SPEC):**
- All inputs have associated `<label>` elements with `htmlFor` matching input `id`
- Errors linked to inputs via `aria-describedby` pointing to error element id
- First field (email) auto-focused via `autoFocus` prop

**Additional:**
- Link below form: "Don't have an account? " (text-zinc-400) + "Register" (text-amber-500 hover:text-amber-400, Link to `/register`)

**What to avoid:**
- Do NOT use gray palette (bg-gray-50, text-gray-900) — use zinc palette per UI-SPEC
- Do NOT use blue for buttons — use amber-500 accent per UI-SPEC
- Do NOT use any form library — plain useState is sufficient
- Do NOT store tokens manually — use useAuthStore().setAuth() only
