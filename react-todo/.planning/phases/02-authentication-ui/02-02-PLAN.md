**Dark theme per UI-SPEC (same as LoginPage for consistency):**
- Outer: `min-h-screen flex items-center justify-center bg-zinc-950`
- Card: `bg-zinc-900 border border-zinc-800 rounded-lg p-8 w-full max-w-md`
- Title: "Create Account" centered, `text-2xl font-bold text-zinc-100 mb-6`
- Text: `text-zinc-100` for labels, `text-zinc-400` for placeholders
- Input borders: `border-zinc-800 bg-zinc-900 text-zinc-100 placeholder-zinc-400`
- Submit button: `bg-amber-500 hover:bg-amber-400 text-zinc-950 font-medium py-3 w-full`
- Link text: `text-amber-500 hover:text-amber-400`
- Error text: `text-sm text-red-500`
- Disabled button: `bg-zinc-800 text-zinc-600 cursor-not-allowed`

**Fields (per D-03 — 5 fields, per D-04 — validation on blur):**
1. Email: `type="email"`, label "Email", placeholder "your@email.com", `min-h-[44px]`
2. Password: `type="password"`, label "Password", placeholder "••••••••", `min-h-[44px]`
3. Confirm Password: `type="password"`, label "Confirm Password", placeholder "••••••••", `min-h-[44px]`
4. First Name: `type="text"`, label "First Name", placeholder "John", `min-h-[44px]`
5. Last Name: `type="text"`, label "Last Name", placeholder "Doe", `min-h-[44px]`
6. Submit button: "Create Account", full-width

**Field order:** Email → Password → Confirm Password → First Name → Last Name → Submit. This groups password fields together for better UX.

**State management:**
- `email`, `password`, `confirmPassword`, `firstName`, `lastName` — controlled inputs
- `errors` — object `{ email?: string, password?: string, confirmPassword?: string, firstName?: string, lastName?: string, general?: string }`
- `isSubmitting` — boolean
- `useNavigate` from react-router

**Validation on blur (per D-04, D-05):**
- Email: required, must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` regex
- Password: required, minimum 6 characters
- Confirm Password: required, must match `password` field value (per D-05 — validates against password on blur)
- First Name: required, minimum 1 character (non-empty after trim)
- Last Name: required, minimum 1 character (non-empty after trim)
- On blur: if field touched and invalid, set error
- On change: clear the specific field's error (per D-12)
- Confirm Password: on blur, if `password !== confirmPassword`, set error to "Passwords do not match." (per UI-SPEC)

**Submit handler:**
1. Prevent default, validate all fields, if invalid return early
2. Set `isSubmitting = true`, clear all errors
3. Call `apiClient.post<LoginResponse>('/auth/register', { email, password, firstName, lastName })` — note: confirmPassword is NOT sent to API (client-side only, per D-03)
4. On success: call `useAuthStore().setAuth({ token, refreshToken, firstName, lastName })` from response, then `navigate('/dashboard')`
5. On 409 Conflict (email already exists): set `errors.email` to "An account with this email already exists." (per UI-SPEC)
6. On other 4xx error: set `errors.general` to the server message if available, otherwise "Registration failed. Please try again."
7. On network error: set `errors.general` to "Unable to connect. Please check your connection and try again." (per D-11, UI-SPEC)
8. On any error: set `isSubmitting = false`

**Submit button state (per D-13):**
- Disabled when `isSubmitting` is true
- Shows spinner when submitting: inline SVG with `animate-spin h-5 w-5 mr-2`
- Button text: "Creating account..." when submitting, "Create Account" otherwise

**Accessibility (per UI-SPEC):**
- All inputs have associated `<label>` elements with `htmlFor` matching input `id`
- Errors linked to inputs via `aria-describedby`
- First field (email) auto-focused via `autoFocus` prop

**Additional:**
- Link below form: "Already have an account? " (text-zinc-400) + "Sign In" (text-amber-500 hover:text-amber-400, Link to `/login`)

**What to avoid:**
- Do NOT send confirmPassword to the API — it's client-side validation only
- Do NOT use gray palette — use zinc palette per UI-SPEC
- Do NOT use blue for buttons — use amber-500 accent
- Do NOT use any form library — plain useState is sufficient
- Do NOT store tokens manually — use useAuthStore().setAuth() only
