Create `docker/Dockerfile` with multi-stage build per RESEARCH.md Pattern 4:

```dockerfile
# Stage 1: Build the Vue application
FROM node:24-alpine AS builder
WORKDIR /app

# Copy package files first for better layer caching
COPY package*.json ./
RUN npm ci

# Copy source and build
COPY . .
RUN npm run build

# Stage 2: Serve with nginx
FROM nginx:alpine

# Copy nginx configuration
COPY docker/nginx.conf /etc/nginx/nginx.conf

# Copy built assets from builder stage
COPY --from=builder /app/dist /usr/share/nginx/html

# Expose port 8080 (nginx default)
EXPOSE 8080

# Use non-root nginx user for security (per RESEARCH.md Security Domain)
USER nginx

CMD ["nginx", "-c", "/etc/nginx/nginx.conf", "-g", "daemon off;"]
```

**Key decisions (per RESEARCH.md):**
- `node:24-alpine` — latest Node for building Vue 3 app
- `nginx:alpine` — 5MB vs 130MB for regular nginx (State of the Art)
- `npm ci` instead of `npm install` — reproducible builds
- `USER nginx` — non-root execution (security control from Phase 1 research)
- `EXPOSE 8080` — nginx default port

Place at `docker/Dockerfile` (not `Dockerfile` at root — the docker/ directory is the build context).

Create `docker/nginx.conf` (and copy to `nginx.conf` at root for build context) per RESEARCH.md Pattern 5:

```nginx
worker_processes auto;
error_log /var/log/nginx/error.log;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;
    sendfile on;
    tcp_nopush on;
    keepalive_timeout 65;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml;

    # Server block
    server {
        listen 8080;
        server_name localhost;
        root /usr/share/nginx/html;
        index index.html;

        # Health check endpoint
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }

        # Assets: cache forever (hashed filenames from Vite build)
        location ^~ /assets/ {
            add_header Cache-Control "public, max-age=31536000, immutable";
            try_files $uri =404;
        }

        # Workbox scripts: cache forever
        location ^~ /workbox- {
            add_header Cache-Control "public, max-age=31536000, immutable";
            try_files $uri =404;
        }

        # API proxy to backend (.NET) — DOCK-03: no CORS needed via proxy
        location /api/ {
            proxy_pass http://host.docker.internal:8080/api/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_connect_timeout 30s;
            proxy_read_timeout 30s;
        }

        # SPA fallback: all other routes serve index.html — fixes routing refresh 404
        location / {
            try_files $uri $uri/ /index.html;
            add_header Cache-Control "public, max-age=0, must-revalidate" always;
        }
    }
}
```

**Copy to root for Docker build context:**
Copy `docker/nginx.conf` to `nginx.conf` at project root so Docker build context can find it.

**Key points (per RESEARCH.md):**
- `try_files $uri $uri/ /index.html` — fixes Pitfall 3 (SPA routing breaks on refresh)
- `location /api/` proxy — DOCK-03: removes need for CORS configuration by proxying backend requests
- `host.docker.internal` — Docker container reaches host machine on Windows/Mac (Linux: may need `--network=host`)
- `gzip` compression — performance optimization
- Health endpoint at `/health` — DOCK-02 verification endpoint

Create `docker-compose.yml` per RESEARCH.md Pattern 6:

```yaml
version: '3.8'
services:
  frontend:
    build:
      context: .
      dockerfile: docker/Dockerfile
    ports:
      - "3000:8080"
    environment:
      - VITE_API_BASE_URL=http://localhost:8080/api/v1
    extra_hosts:
      - "host.docker.internal:host-gateway"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s
```

**Key decisions:**
- Port 3000 on host → 8080 in container (nginx default)
- `VITE_API_BASE_URL` env var for frontend to know backend URL
- `extra_hosts` — enables `host.docker.internal` on Linux (on Windows/Mac this is automatic)
- Healthcheck verifies container is responding

**Verify the container builds and serves:**
```bash

docker-compose up -d

curl http://localhost:3000/health

```

**For local dev without Docker:**
The Vite dev server (`npm run dev`) handles SPA routing and proxies `/api` via vite.config.ts proxy option. The docker-compose is for containerized local dev and production-like testing.
