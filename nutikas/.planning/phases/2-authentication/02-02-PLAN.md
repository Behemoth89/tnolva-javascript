<feature>
  <name>Axios API Client with JWT Interceptors</name>
  <files>src/api/index.ts, tests/api/index.spec.ts</files>
  <behavior>
    1. Request interceptor reads JWT from auth store and injects as Authorization header
    2. Response interceptor catches 401 errors and triggers token refresh
    3. If refresh succeeds: new JWT stored, original request retried with new token
    4. If refresh fails: tokens cleared, user redirected to /login
    5. Multiple concurrent 401s share one refresh call (queue pattern)

    Cases:
    - Normal request → JWT injected, 200 returned
    - Expired JWT → 401 → refresh → retry with new JWT → 200
    - Refresh token expired → 401 on refresh → clear tokens → redirect to login
    - Multiple concurrent 401s → only one refresh call, others wait
  </behavior>
  <implementation>
1. Create src/api/index.ts with:
   - Base Axios instance (baseURL from VITE_API_BASE_URL/env)
   - Request interceptor: inject Authorization: Bearer {jwt}
   - Response interceptor: on 401, call refresh endpoint, update store, retry request
   - isRefreshing flag + refreshSubscribers queue to handle concurrent 401s

2. Create tests/api/index.spec.ts with Vitest tests covering:
   - Request interceptor injects JWT when present
   - Response interceptor triggers refresh on 401
   - Failed refresh clears tokens and redirects
   - Concurrent 401s share one refresh call
  </implementation>
</feature>
