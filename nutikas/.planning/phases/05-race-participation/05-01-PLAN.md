**State:**
- `raceState: RaceState` тАФ initialized as `{ startDT: null, finishDT: null, score: 0, bonus: 0, penalty: 0, finalScore: 0, scannedCPIds: [] }`
- `loading: ref&lt;boolean&gt;`
- `error: ref&lt;string | null&gt;`

**Computed:**
- `isStarted` тАФ derived from `raceState.startDT !== null`
- `isFinished` тАФ derived from `raceState.finishDT !== null`

**Actions:**
- `submitScan(checkPointId: string, userTeamId: string): Promise&lt;MarkingSubmitResult&gt;`
  1. Call `submitMarking({ checkPointId, userTeamId })`
  2. Build a `MarkingSubmitResult` from the response
  3. Detect `isAlreadyScanned`: check if `scannedCPIds` already contains this `checkPointId`
  4. Update `raceState` from `response.result` if result is not null (startDT, finishDT, score, bonus, penalty, finalScore)
  5. If NOT already scanned, append `checkPointId` to `scannedCPIds`
  6. Return the `MarkingSubmitResult`

- `resetRace(): void` тАФ reset `raceState` to initial values, clear `scannedCPIds`

- `loadRaceState(activation: UserTeamActivation): void` тАФ restore race state from a previously fetched activation (pre-fill scannedCPIds from activation.markings if available)

Create `src/composables/useToast.ts` using the provide/inject pattern:

**Types:**
- `ToastType = 'success' | 'error' | 'info'`
- `Toast = { id: number, message: string, type: ToastType, autoDismiss: boolean }`

**Internal:**
- `TOAST_SYMBOL = Symbol('toast')`
- `createToast()` function that returns `{ toasts, show, dismiss, success, error, info }`
  - `toasts: ref&lt;Toast[]&gt;`
  - `show(message, type, autoDismiss)` тАФ adds toast, sets auto-dismiss timeout if autoDismiss=true (3000ms)
  - `dismiss(id)` тАФ removes toast by id
  - `success(msg)` тАФ shorthand for `show(msg, 'success', true)`
  - `error(msg)` тАФ shorthand for `show(msg, 'error', true)`
  - `info(msg)` тАФ shorthand for `show(msg, 'info', true)`
  - Auto-dismiss uses `setTimeout(() => dismiss(id), 3000)`

**provideToast()** тАФ exported, wraps `provide(TOAST_SYMBOL, createToast())`
**useToast()** тАФ exported, returns `inject(TOAST_SYMBOL) as ReturnType&lt;typeof createToast&gt;`

Per D-11: auto-dismiss timing is a user preference тАФ the composable accepts `autoDismiss` param per-call. The user preference toggle (auto vs manual) is handled at the component level.
