Create `src/views/RaceView.vue` — the combined race participation screen.

**Route params:**
- `contestId: string` — from `/race/:contestId/:userTeamId`
- `userTeamId: string` — from `/race/:contestId/:userTeamId`

**Composition:**
- Import `useRaceStore` for raceState, submitScan, resetRace, loadRaceState
- Import `useTeamStore` for activeTeam data
- Import `useToast` and call `provideToast()` at app level (or in App.vue)
- Import `TeamInfoCard`, `ScoreCard`, `RaceScanner`, `RaceScorePanel` components

**State machine (pre-race → active → post-race):**
- `racePhase: ref&lt;'pre' | 'active' | 'post'&gt;` — derived from raceState
  - `'pre'`: `startDT === null` — show team info card + scanner preview ready
  - `'active'`: `startDT !== null && finishDT === null` — scanner active + live score
  - `'post'`: `finishDT !== null` — show final breakdown with RaceScorePanel

**onMounted:**
1. Call `teamStore.loadActiveTeam()` to get activeTeam from IndexedDB
2. Call `raceStore.loadRaceState(teamStore.currentTeamDetail)` to pre-fill from any existing activation
3. If `activeTeam` has no teamId, redirect or show "no active team" error

**Scanner integration:**
- RaceScanner emits `scan-success` → update score display via raceStore
- RaceScanner emits `scan-error` → toast already handled inside scanner, no extra action needed
- RaceScanner emits `camera-error` → show error toast

**Refresh handling:**
- ScoreCard emits `refresh` → call `teamStore.fetchTeamDetail(props.userTeamId)` then update raceState from response
- Also refresh position via `contestApi.getContestResults(contestId)`

**Combined layout (per D-03/D-04, two-panel layout):**
- Left panel (or top on mobile): TeamInfoCard + ScoreCard (pre-race shows team info + scanner preview; active shows live score + position)
- Right panel (or bottom on mobile): RaceScanner camera + manual entry
- Post-race: RaceScorePanel replaces ScoreCard

**Template structure:**
```
&lt;div class="race-view"&gt;
  &lt;!-- Loading/error states from ContestDetailView pattern --&gt;
  
  &lt;div v-if="racePhase === 'pre'" class="pre-race"&gt;
    &lt;TeamInfoCard :teamName="..." :contestClassName="..." :memberNames="..." /&gt;
    &lt;RaceScanner ... /&gt;
  &lt;/div&gt;
  
  &lt;div v-else-if="racePhase === 'active'" class="active-race"&gt;
    &lt;div class="score-panel"&gt;
      &lt;ScoreCard ... /&gt;
    &lt;/div&gt;
    &lt;div class="scanner-panel"&gt;
      &lt;RaceScanner ... /&gt;
      &lt;!-- Floating refresh FAB (D-09) --&gt;
    &lt;/div&gt;
  &lt;/div&gt;
  
  &lt;div v-else-if="racePhase === 'post'" class="post-race"&gt;
    &lt;RaceScorePanel :startDT="..." :finishDT="..." :score="..." :bonus="..." :penalty="..." :finalScore="..." /&gt;
    &lt;RaceScanner ... /&gt;  &lt;!-- still show scanner in case user wants to verify scans --&gt;
  &lt;/div&gt;
&lt;/div&gt;
```

**Styling:**
- Use CSS grid or flexbox for the two-panel layout
- On mobile, stack vertically (scanner on top, score below OR score on top, scanner below per D-03)
- Scanner should be visually prominent — fills most of the screen
- Score panel always visible during active race (D-04)
