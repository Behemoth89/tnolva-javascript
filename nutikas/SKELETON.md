## Technology Stack

| Category | Choice | Rationale |
|----------|--------|-----------|
| Framework | Vue 3 + TypeScript | User-specified, mobile-first |
| Build tool | Vite 8.x | Fast HMR, standard for Vue 3 |
| State management | Pinia 2.x | TypeScript-native, later phases |
| Routing | Vue Router 4.x | SPA client-side routing |
| CSS | Tailwind CSS 3.x | Mobile-first utility CSS |
| PWA | vite-plugin-pwa + Workbox | Zero-config PWA generation |
| Icons | pwa-assets-generator | Generates all required sizes |
| API client | Axios | Later phase — for API calls |
| Offline storage | Dexie (IndexedDB) | Later phase — offline queue |
| Testing | Vitest | Later phase |
| Linting | ESLint 9.x + Prettier 3.x | Code quality |
| Container | Docker + nginx:alpine | Production serve |
| API proxy | nginx (no CORS) | Proxy removes CORS need |

## Deployment

| Concern | Decision |
|---------|----------|
| Container image | `nutikas:latest` from `docker/Dockerfile` |
| Port mapping | Host 3000 → Container 8080 (nginx) |
| API proxy | nginx proxies `/api/` → `http://host.docker.internal:8080/api/` |
| Dev port | 5173 (Vite dev server) |
| Production serve | nginx:alpine in Docker |

## API Conventions

- Base URL: `http://localhost:8080/api/v1` (dev), `http://localhost:3000/api/v1` (Docker)
- Environment variable: `VITE_API_BASE_URL`
- Proxy via nginx — no CORS configuration needed
- All endpoints return JSON, times in ISO 8601 UTC
- Property names: camelCase on the wire
- See: `devhelp/api-spec.json` (full OpenAPI spec)

## Key Implementation Notes

### PWA
- `registerType: 'autoUpdate'` — service worker auto-updates
- `cleanupOutdatedCaches: true` — removes old caches on update
- ReloadPrompt.vue handles update notification (PWA-03)

### Docker
- `host.docker.internal` enables container → host machine access (Windows/Mac)
- Linux: may need `--network=host` or explicit IP
- Health endpoint: `/health` returns "healthy"

### API Proxy
- nginx proxies `/api/` to backend — browser never makes direct cross-origin calls
- No CORS configuration needed on frontend or backend
