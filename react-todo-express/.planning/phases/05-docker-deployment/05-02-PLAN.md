  <read_first>
    - .planning/phases/05-docker-deployment/05-CONTEXT.md — locked decisions D-01, D-04
    - package.json — project name is react-todo
  </read_first>
  <action>
    Create nginx.conf file at project root with the following configuration:

    **Server block:**
    - listen 80
    - server_name localhost
    - root /usr/share/nginx/html
    - index index.html

    **Location / block (critical for SPA routing):**
    - try_files $uri $uri/ /index.html
    - This is the key directive: when nginx can't find a matching file (like /dashboard or /settings), it serves index.html so React Router can handle the route client-side
    - Without this, refreshing on any route other than / returns 404

    **Gzip compression for production assets:**
    - Enable gzip for text/html, application/javascript, text/css, application/json
    - This is standard for SPA serving — reduces bundle transfer size

    **Security headers (optional but good for school project):**
    - Add X-Content-Type-Options: nosniff
    - Add X-Frame-Options: SAMEORIGIN

    The Dockerfile (plan 01) copies this file to /etc/nginx/conf.d/default.conf in the nginx container.
  </action>
  <acceptance_criteria>
    - nginx.conf exists in project root
    - Contains "listen 80"
    - Contains "try_files $uri $uri/ /index.html" (exact string — critical for SPA routing)
    - Contains "root /usr/share/nginx/html"
    - Contains "index index.html"
    - Contains "server_name"
  </acceptance_criteria>
  <verify>
    <automated>test -f nginx.conf && grep -q 'try_files' nginx.conf && grep -q 'index.html' nginx.conf && grep -q 'listen 80' nginx.conf && echo "PASS" || echo "FAIL"</automated>
  </verify>
  <done>nginx.conf exists with try_files directive for SPA routing fallback to /index.html, listening on port 80</done>
